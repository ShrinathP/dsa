/*
Problem Statement
Given:
- coins[] (infinite supply of each coin)
- target

👉 Find minimum number of coins to make target
👉 Return -1 if not possible

Striver Explanation
*/

/*
As there is infinite supply of coins
This is UNBOUNDED:
take → stay at same index (i)
notTake → move to i-1

NOTE IMP 
why greedy doesnt work here ? i remember some coin problems where greedy worked
Greedy works in canonical coin systems
- Indian currency: 1, 2, 5, 10, 20, 50, …
- US coins: 1, 5, 10, 25
Example
coins = [1, 5, 10], target = 18
Greedy:
10 → 5 → 1 → 1 → 1  → 5 coins
Optimal? Yes.

For arbitary structure
coins = [1, 3, 4], target = 6
Greedy:
4 → 1 → 1  → 3 coins
Optimal:
3 → 3  → 2 coins
👉 Greedy gets stuck in a locally optimal choice (4)
but misses a globally optimal solution (3+3)

Greedy always assumes
choosing the largest coin now → leads to optimal solution
But that’s not always true unless the coin system has special structure.


IMP - Not using INT.MAX as any of the take function, 1 + INT.MAX will be overflow
Top down recursion
Steps:
1. recurMethod(index, target, coins arr)
    1.1 base case
    if index == 0, if target is divisible by coins[0] i.e. no remainder, add the divisor value(i.e. no of coins)
    if(index == 0) {
        if(target % coins[0] == 0) return target/coins[0];
        return 1e9;
    }
    1.2 notTake case -> recurMethod(index - 1, target, coins);
    1.3 take case -> init (int)1e9, if (coins[ind] <= target) then take it
        take = (int)1e9;
        if (coins[ind] <= target) {
            take = 1 + recurMethod(index - 1, target - coins[ind], coins);
        }
    1.4 return Math.min(take, notTake)
*/


class Solution {

    public int minCoins(int[] coins, int n, int target) {
        int ans = helper(n - 1, target, coins);
        return (ans >= (int)1e9) ? -1 : ans;
    }

    private int helper(int index, int target, int[] coins) {

        // Base case
        if (index == 0) {
            if (target % coins[0] == 0) {
                return target / coins[0];
            }
            return (int)1e9; // impossible
        }

        int notTake = helper(index - 1, target, coins);

        int take = (int)1e9;
        if (coins[index] <= target) {
            // stay at same index (unbounded)
            take = 1 + helper(index, target - coins[index], coins);
        }

        return Math.min(take, notTake);
    }
}
